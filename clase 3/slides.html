XMLHttpRequest Clásico · 2001

Basado en callbacks. Importante para entender la historia.


const xhr = new XMLHttpRequest();

// 1. Configurar: método, URL, asíncrono (true = no bloquea)
xhr.open('GET', 'api.php?id=1', true);

// 2. Qué hacer cuando llega la respuesta
xhr.onload = function () {
    if (xhr.status >= 200 && xhr.status < 300) {
        const datos = JSON.parse(xhr.responseText);
        console.log(datos);
    } else {
        console.error('Error HTTP:', xhr.status);
    }
};

// 3. Errores de red (sin conexión, CORS bloqueado)
xhr.onerror = () => console.error('Error de red');

// 4. Disparar la petición
xhr.send();

Estados de XHR — readyState

Ciclo de vida de la petición

ValorConstanteSignificado
0UNSENTopen() no fue llamado
1OPENEDopen() fue llamado
2HEADERS_RECEIVEDSend() llamado, headers OK
3LOADINGDescargando el body
4 DONE Respuesta completa ✓
onload se dispara solo cuando readyState === 4. En código legacy verás el más verboso onreadystatechange.

Fetch API Moderno · 2015

Basado en Promesas. Mucho más legible que XHR.


fetch('api.php?id=1')
    .then(response => {
        if (!response.ok) throw new Error(`HTTP ${response.status}`);
        return response.json();  // retorna otra Promesa
    })
    .then(datos => {
        console.log(datos.nombre);
    })
    .catch(err => console.error('Error:', err.message));
⚠️ Trampa clásica: fetch() NO rechaza la promesa cuando el servidor responde con un error HTTP (404, 500, etc.). Solo rechaza en errores de red. Siempre verificar response.ok.

El objeto Response

Lo que devuelve la primera promesa de fetch()

Propiedades

response.oktrue si 200–299
response.statusCódigo HTTP
response.headersCabeceras recibidas

Métodos (→ Promesa)

response.json()Parsea como JSON
response.text()Lee como string
response.blob()Para archivos/imgs
⚠️ El body solo se puede leer una vez. Si llamás response.json(), ya no podés llamar response.text().

¿Qué es una Promesa?

⏳
pending
esperando respuesta
✅
fulfilled
resuelta con valor
❌
rejected
rechazada con error

const promesa = fetch('api.php');  // → pending…

promesa
    .then(res => res.json())   // si se resuelve → .then()
    .catch(err => ...)         // si se rechaza  → .catch()
    .finally(() => ...);       // siempre (para cleanup/loading)

async / await Recomendado

Syntactic sugar sobre promesas. Mismo comportamiento, código más legible.

Con .then()


fetch('api.php')
  .then(res => res.json())
  .then(data => {
    console.log(data);
  })
  .catch(err => {
    console.error(err);
  });

Con async/await ✓


async function cargar() {
  try {
    const res  = await fetch('api.php');
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
await solo puede usarse dentro de una función async. Los errores se capturan con try / catch.

POST con fetch Enviar datos


async function crearEstudiante(datos) {
    const res = await fetch('api.php', {
        method:  'POST',
        headers: { 'Content-Type': 'application/json' },
        body:    JSON.stringify(datos),  // objeto JS → string JSON
    });

    if (!res.ok) {
        const error = await res.json();
        throw new Error(error.message);
    }

    return await res.json();
}
Siempre acompañar un body JSON con el header Content-Type: application/json para que PHP sepa cómo leerlo.

Formulario sin recarga

El patrón más usado del curso


document.getElementById('mi-form')
  .addEventListener('submit', async (e) => {

    e.preventDefault();   // 1. Evitar recarga de página
    const btn = e.target.querySelector('[type=submit]');

    // 2. Leer campos del formulario como objeto JS
    const payload = Object.fromEntries(new FormData(e.target));

    btn.disabled = true;  // 3. Estado de carga
    try {
      const res  = await fetch('api.php', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body:   JSON.stringify(payload),
      });
      if (!res.ok) throw new Error((await res.json()).message);
      e.target.reset();   // 4. Limpiar solo si fue exitoso
    } catch (err) {
      alert('Error: ' + err.message);
    } finally {
      btn.disabled = false; // 5. Siempre reactivar botón
    }
  });

PHP recibe peticiones AJAX

El servidor no distingue si la petición viene de un formulario clásico o de fetch 


<?php
header('Content-Type: application/json');

$method = $_SERVER['REQUEST_METHOD'];

// GET clásico → parámetros en $_GET
// POST desde form HTML → datos en $_POST

if ($method === 'POST') {
    // fetch() con JSON body → leer el input crudo
    $body = json_decode(file_get_contents('php://input'), true);
    // $body es ahora un array PHP con los datos enviados
}
Form HTML clásico
Datos en $_POST['campo']
fetch() con JSON body
Leer con php://input + json_decode()
