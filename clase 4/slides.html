¿Qué es JSON?

JavaScript Object Notation · RFC 8259 · ECMA-404

Historia

  • Nació de los objetos JS (~2001)
  • Douglas Crockford lo popularizó
  • Independiente del lenguaje hoy
  • RFC 8259 (2017) es el estándar

Ventajas

  • Ligero (sin etiquetas redundantes)
  • Legible por humanos
  • Soporte nativo en el navegador
  • Anidamiento libre

Mismo dato — JSON vs XML:

// JSON — 60 bytes
{
  "id": 1,
  "nombre": "Laptop",
  "precio": 850
}

<producto>
  <id>1</id>
  <nombre>Laptop</nombre>
  <precio>850</precio>
</producto>

Tipos de datos válidos en JSON

JSON admite exactamente 6 tipos. Ni más, ni menos.

string"Hola mundo"Comillas dobles OBLIGATORIAS
number42 · 3.14 · -7 · 1.5e3Sin comillas, sin NaN/Infinity
booleantrue · falseMinúsculas. No True, no FALSE
nullnullMinúsculas. No NULL, no undefined
array[1, "dos", true, null]Cualquier tipo, incluidos objetos
object{"clave": "valor"}Claves siempre strings con comillas dobles
No existen en JSON: undefined · function · Symbol · Date · NaN · Infinity · comentarios

Reglas estrictas de sintaxis

❌ JSON INVÁLIDO

// Clave sin comillas
{ nombre: "Juan" }

// Comillas simples
{ "nombre": 'Juan' }

// Trailing comma
{
  "a": 1,
  "b": 2,
}

// Comentario
{
  // nombre del usuario
  "nombre": "Ana"
}

// Valores JS no soportados
{ "fn": () => 42 }
{ "undef": undefined }
{ "nan": NaN }

✅ JSON VÁLIDO

{
  "nombre": "Juan",
  "edad": 20,
  "activo": true,
  "beca": null,
  "cursos": [
    "Matemáticas",
    "Física"
  ],
  "direccion": {
    "ciudad": "Cochabamba",
    "pais": "Bolivia"
  }
}
Recordá: JSON es más estricto que un objeto literal de JavaScript.

JSON.parse() string → valor JS

Uso básico

const json = '{"nombre":"Ana","edad":22}';
const obj  = JSON.parse(json);

console.log(obj.nombre); // "Ana"
console.log(obj.edad);   // 22 ← number, no string
console.log(typeof obj); // "object"

// ⚠️ Siempre usar try/catch con datos externos
try {
    const datos = JSON.parse(inputDelUsuario);
} catch (e) {
    // e es un SyntaxError
    console.error('JSON inválido:', e.message);
}

El parámetro reviver

// Transformar valores al parsear
const json = '{"fecha":"2025-03-11","precio":"850"}';

const obj = JSON.parse(json, (clave, valor) => {
    if (clave === 'fecha')  return new Date(valor);
    if (clave === 'precio') return parseFloat(valor);
    return valor;
});

console.log(obj.fecha instanceof Date); // true
console.log(typeof obj.precio);         // "number"
Casos de uso:
  • Leer datos de localStorage
  • Procesar respuesta de fetch
  • Deserializar config JSON

JSON.stringify() valor JS → string

JSON.stringify(valor, replacer, space) — los 3 parámetros

const p = { id:1, nombre:'Laptop', precio:850, password:'secret', stock:12 };

// Sin opciones — compacto
JSON.stringify(p);
// → {"id":1,"nombre":"Laptop","precio":850,"password":"secret","stock":12}

// Con space — legible (2 espacios)
JSON.stringify(p, null, 2);
// {
//   "id": 1,
//   "nombre": "Laptop", ...

// Replacer array — solo esos campos (lista blanca)
JSON.stringify(p, ['id', 'nombre', 'precio']);
// → {"id":1,"nombre":"Laptop","precio":850}
// password y stock no aparecen

// Replacer función — control total
JSON.stringify(p, (k, v) => {
    if (k === 'password') return undefined; // omitir
    return v;
});

¿Qué omite stringify?

Objeto con valores problemáticos

const obj = {
  nombre:    'Test',
  funcion:   () => 'hola',   // función
  indefinido: undefined,      // undefined
  simbolo:   Symbol('x'),    // Symbol
  fecha:     new Date(),     // Date
  nan:       NaN,            // NaN
  infinito:  Infinity,       // Infinity
  nulo:      null,           // null (sí aparece)
};

Resultado de stringify

{
  "nombre": "Test",
  "fecha":  "2025-03-11T14:00:00.000Z",
  "nan":    null,
  "infinito": null,
  "nulo":   null
}
// funcion → OMITIDA
// indefinido → OMITIDA
// simbolo → OMITIDA
Deep clone con JSON:
JSON.parse(JSON.stringify(obj))
Funciona para datos simples. Pierde funciones, Date real y undefined. Para casos complejos usar structuredClone(obj).

Ejemplo completo

Ciclo completo: objeto → stringify → transferencia → parse → modificar → stringify de nuevo 

// 1. Objeto JavaScript original
const carrito = {
    items: [{ id: 1, nombre: 'Laptop', cantidad: 1 }],
    total: 850.00,
};

// 2. Serializar para enviar (o guardar en localStorage)
const jsonString = JSON.stringify(carrito);
localStorage.setItem('carrito', jsonString);

// 3. Recuperar y deserializar (puede ser en otra función/página)
const recuperado = JSON.parse(localStorage.getItem('carrito'));
// recuperado es un nuevo objeto, independiente del original
recuperado.items.push({ id: 2, nombre: 'Mouse', cantidad: 2 });
recuperado.total += 25.50;

// 4. Volver a serializar para enviar al servidor
const payload = JSON.stringify(recuperado);
// → {"items":[{"id":1,...},{"id":2,...}],"total":875.5}

// 5. El servidor responde JSON → parsear la respuesta
const res = await fetch('/api/carrito', { method:'POST', body: payload,
    headers: {'Content-Type':'application/json'} });
const confirmacion = await res.json(); // fetch tiene .json() que hace JSON.parse internamente

¿Cuándo usar cada formato?

JSON application/json

  • APIs REST (casi siempre)
  • Peticiones AJAX con fetch
  • Datos estructurados anidados
  • LocalStorage, sessionStorage

URL Encoded x-www-form-urlencoded

  • Formularios HTML sin JS
  • Datos simples clave=valor
  • Parámetros en la URL (GET)

Multipart multipart/form-data

  • Formularios con subida de archivos
  • Imágenes, PDFs, documentos
  • Mezcla de texto y binarios

XML application/xml

  • Integración con sistemas legacy
  • SOAP / XML-RPC
  • Documentos con metadatos ricos

json_encode() en PHP

Uso básico y opciones

$producto = [
    'id'     => 1,
    'nombre' => 'Laptop HP 15"',
    'precio' => 850.00,
    'tags'   => ['electrónica', 'oferta'],
];

// Compacto (default)
echo json_encode($producto);

// Legible + unicode + slashes
echo json_encode(
    $producto,
    JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES
);

// Lanzar excepción en error (PHP 7.3+)
$json = json_encode(
    $datos,
    JSON_THROW_ON_ERROR
);

Opciones clave

OpciónEfecto
JSON_PRETTY_PRINTIndenta 4 espacios
JSON_UNESCAPED_UNICODEMantiene é,ñ,ó sin \u
JSON_UNESCAPED_SLASHESMantiene / sin \/
JSON_THROW_ON_ERRORLanza JsonException
Siempre enviar la cabecera antes de json_encode:
header('Content-Type: application/json; charset=utf-8');

json_decode() en PHP

Dos modos: array vs objeto

$json = '{"nombre":"Ana","cursos":["Mate","Física"]}';

// Con true → array asociativo (recomendado para APIs)
$arr = json_decode($json, true);
echo $arr['nombre'];      // "Ana"
echo $arr['cursos'][0];   // "Mate"

// Sin true → objeto stdClass
$obj = json_decode($json);
echo $obj->nombre;        // "Ana"
echo $obj->cursos[0];     // "Mate"

// Siempre verificar errores
if (json_last_error() !== JSON_ERROR_NONE) {
    $msg = json_last_error_msg();
    // Manejar el error...
    http_response_code(400);
    echo json_encode(['error' => $msg]);
    exit;
}

Constantes de error

ConstanteDescripción
JSON_ERROR_NONESin errores (0)
JSON_ERROR_SYNTAXSintaxis inválida
JSON_ERROR_UTF8Caracteres UTF-8 mal formados
JSON_ERROR_DEPTHAnidamiento muy profundo
Con JSON_THROW_ON_ERROR podés usar try/catch con JsonException en lugar de verificar manualmente.

Recibir JSON en PHP

Con Content-Type: application/json, el body NO llega en $_POST. Hay que leerlo desde php://input.

Cliente (JavaScript)

// fetch envía JSON en el body
await fetch('api.php', {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify({
        nombre: 'Laptop',
        precio: 850,
    }),
});

Servidor (PHP)

header('Content-Type: application/json');

// 1. Leer el stream raw de entrada
$raw = file_get_contents('php://input');

if (empty($raw)) {
    http_response_code(400);
    echo json_encode(['error' => 'Body vacío']);
    exit;
}

// 2. Parsear
$data = json_decode($raw, true);
if (json_last_error() !== JSON_ERROR_NONE) {
    http_response_code(400);
    echo json_encode(['error' => json_last_error_msg()]);
    exit;
}

// 3. Usar los datos
echo json_encode(['nombre' => $data['nombre']]);
exit;

Anti-patrones comunes

❌ No verificar si JSON.parse lanzó error
const d = JSON.parse(textoDelusuario); ✅ try { JSON.parse(...) } catch(e) { /* manejar */ }
❌ No enviar Content-Type en PHP
echo json_encode($data); // sin header ✅ header('Content-Type: application/json'); echo json_encode(...);
❌ Leer body JSON desde $_POST
$d = $_POST['nombre']; // nunca llega con application/json ✅ $data = json_decode(file_get_contents('php://input'), true);
❌ Usar comillas simples en JSON manual
{'nombre': 'Juan'} ✅ {"nombre": "Juan"}
❌ Usar deep clone con JSON para objetos con Date/funciones
JSON.parse(JSON.stringify(objConFecha)) // Date → string ✅ structuredClone(obj) // o _.cloneDeep(obj)
